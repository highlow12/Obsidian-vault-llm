## 📖 What Is This Thing?

**Obsidian Vault LLM (OVL)** is an Obsidian desktop plugin that indexes your vault's markdown notes into vector embeddings, then lets you ask an LLM about them.

In short: you dump all your thoughts into Obsidian for years, forget 95% of it, and then ask an AI to find the 5% you actually need.

- 📝 "Summarize last week's meeting notes" → searches relevant notes and generates an answer
- 🔍 Semantically indexes your entire vault so you can search for vibes, not just keywords
- 💾 Saves conversations as markdown notes with automatic topic separation
- 🤖 Supports Google Gemini, Anthropic Claude, OpenAI, Ollama (local), and custom APIs

All vault data is stored **locally only**. Only the necessary context is sent to the LLM API. We pinky-promise.

### 🔍 Advanced Search Query Syntax

You can use special syntax to tell the plugin exactly where to look. Think of it as talking to a very literal-minded librarian.

#### Folder / File Filters

| Syntax | Description |
|--------|-------------|
| `folder:docs` | Search only files in `docs` folder |
| `-folder:private` | Exclude `private` folder (we don't ask questions) |
| `path:projects/2024` | Search only files under specified path |
| `file:guide.md` | Search only a specific file |

#### Tag Filters

| Syntax | Description |
|--------|-------------|
| `tag:rag` or `tag:#rag` | Notes with `rag` tag only |
| `-tag:archive` | Exclude archived notes (out of sight, out of mind) |
| `tag:rag tag:llm` | Notes with any of the tags (OR condition) |

#### Frontmatter Filters

| Syntax | Description |
|--------|-------------|
| `[status]` | Notes that have a `status` property |
| `[status:done]` | Notes where `status` equals `done` (the dream) |
| `[priority:>3]` | Notes where `priority` is greater than 3 |
| `date>=2024-01-01` | Notes modified after 2024-01-01 |

#### Content / Task Filters

| Syntax | Description |
|--------|-------------|
| `content:keyword` | Chunks containing a specific word |
| `task:keyword` | Task items (`- [ ]`, `- [x]`) |
| `task-todo:keyword` | Incomplete tasks — the graveyard of good intentions |
| `task-done:keyword` | Completed tasks — actual achievements |

#### Other Advanced Filters

| Syntax | Description |
|--------|-------------|
| `-keyword` | Exclude a specific word from results |
| `/regex/` | For people who speak regex fluently |
| `match-case:keyword` | When you really care about capitalization |

## 🚀 Usage (The Part You Actually Need)

### Step 1: Installation

#### Manual Install (The Classic)
1. Download the latest `main.js`, `manifest.json`, `styles.css` from [Releases](https://github.com/highlow12/Obsidian-vault-llm/releases).
2. Copy them to your vault's plugin folder:
   ```
   <vault>/.obsidian/plugins/obsidian-vault-llm/
   ```
3. Obsidian → Settings → Community Plugins → Enable "Obsidian Vault LLM"

#### Build from Source (For The Brave)
```bash
npm install        # pray this works
npm run build      # more praying
```

Copy the generated files. You know the drill.

> **Requirements**: Obsidian 1.5.0+, Desktop only. Sorry, mobile users. We can't help you.

---

### Step 2: Configuration

Obsidian → Settings → **OVL Settings** tab. It's like a cockpit, but less impressive.

#### LLM API Settings

| Setting | Description |
|---------|-------------|
| API Provider | Gemini / Claude / OpenAI Compatible / Ollama / Custom |
| API URL | Chat endpoint URL (auto-filled when provider is selected) |
| API Key | Provider API key — **shown as ●●●●● because we're fancy like that** |
| Model | LLM model name |
| Title Model | Separate (cheaper?) model for title generation |
| System Prompt | Tell the AI who it is and what it should do. Go wild. |
| Default Output Folder | Where saved conversations go to live |

**Google Gemini Example**
1. Select `Google Gemini` as provider
2. Enter your [Google AI Studio](https://aistudio.google.com/) API key
3. Click **Load Gemini Model List** → pick the one that sounds coolest

**Anthropic Claude Example**
1. Select `Claude (Anthropic)` as provider
2. Enter your [Anthropic Console](https://console.anthropic.com/) API key (it starts with `sk-ant-`)
3. Enter model name (e.g., `claude-sonnet-4-5`) — Claude will appreciate a good sonnet pun

**Ollama (Local) Example**
1. Install [Ollama](https://ollama.com) and pull a model: `ollama pull llama3.1`
2. Select `Ollama (Local)` as provider (URL auto-filled to `localhost`)
3. Enter `llama3.1` in the model field and pretend it's ChatGPT

#### Vector Indexing Settings (for RAG, a.k.a. The Smart Search)

| Setting | Default | Description |
|---------|---------|-------------|
| Enable Indexing | OFF | Must be turned ON or nothing works. Surprising, right? |
| Chunk Size | 400 | How big each piece of your notes gets. Like chopping vegetables. |
| Chunk Overlap | 50 | How much each chunk overlaps with its neighbor. For context. |
| Top-K | 8 | How many results to retrieve. More = slower but better. Probably. |
| Search Similarity Threshold | 0.03 | Very low. We're not picky. |
| Save Similarity Threshold | 0.75 | How different topics need to be to get their own note. |

---

### Step 3: Start Chatting (Finally)

1. Click the **message icon** in the left ribbon or run **"Open OVL Chat"** from the command palette
2. Type your question and press **Send** or **Ctrl+Enter**
3. Wait. Then marvel at the power of pretrained transformer models.

### Vault Indexing & RAG Search

1. Enable **Indexing** in settings (seriously, don't forget this)
2. Click **Start Full Embedding** (go make a coffee, it'll take a while)
3. Use **Vault Search Answer** in the chat to ask questions about your notes

### Saving & Topic Separation

Click the **Save** button in the chat panel header:
- Conversation → markdown note. Magic.
- Topic Separation AI splits it by topic automatically (works best with 4+ turns — it needs enough context to get confused about the subject, just like you do)

## 🙋 FAQ

**Q: Do I need a paid API?**  
A: You need an API key. Gemini has a free quota (for now). Ollama is free and local. OpenAI will quietly charge your card.

**Q: Can I use it offline?**  
A: Yes! With local embedding (HuggingFace) + Ollama, you can be fully offline and feel smug about it.

**Q: Is my data safe?**  
A: It lives locally on your machine. Only what you query gets sent to the AI. But honestly, you're trusting us, a random hobby project, and whatever AI provider you chose. That's on you, champ.

**Q: How long does initial indexing take?**  
A: ~1–2 minutes for 100 files. ~10–20 minutes for 1000 files. Time to contemplate your note-taking habits.

**Q: Topic separation isn't working well.**  
A: It needs 4+ turns and clear topic changes. If your entire conversation is about one thing, the AI correctly decides there's only one topic. That's a feature, not a bug.

**Q: Something broke.**  
A: Check the error log at `.obsidian/plugins/obsidian-vault-llm/error.log`. Then open an issue. Then wait. We'll get to it eventually, probably.
