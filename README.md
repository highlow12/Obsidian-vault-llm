## 📖 About

**Obsidian Vault LLM (OVL)** is an Obsidian desktop plugin that indexes your vault's markdown notes into vector embeddings, letting you ask natural language questions and get grounded answers from your own notes via LLM.

- 📝 "Summarize last week's meeting notes" → searches relevant notes and generates an answer
- 🔍 Semantically indexes your entire vault for meaning-aware search
- 💾 Saves conversations as markdown notes with automatic topic separation
- 🤖 Supports Google Gemini, Anthropic Claude, OpenAI, Ollama (local), and custom APIs

All vault data is stored **locally only**. Only the necessary context is sent to the LLM API.

### 📚 Vector Indexing & RAG Search
- Chunk-based embedding of all markdown files in the vault
- **Auto re-indexing**: File create/edit/delete/rename triggers automatic update (100ms debounce)
- **Full vault embedding** or **new files only** options
- Cosine similarity Top-K vector search
- Frontmatter, tag, and internal link parsing
- JSON-based local metadata/vector store

### 🔍 Advanced Search Query Syntax

Use filter syntax in the search input to precisely control search scope.

#### Folder / File Filters

| Syntax | Description |
|--------|-------------|
| `folder:docs` | Search only files in `docs` folder |
| `-folder:private` | Exclude `private` folder |
| `path:projects/2024` | Search only files under specified path |
| `file:guide.md` | Search only a specific file |

#### Tag Filters

| Syntax | Description |
|--------|-------------|
| `tag:rag` or `tag:#rag` | Notes with `rag` tag only |
| `-tag:archive` | Exclude notes with `archive` tag |
| `tag:rag tag:llm` | Notes with any of the tags (OR condition) |

#### Frontmatter Filters

| Syntax | Description |
|--------|-------------|
| `[status]` | Notes that have a `status` property |
| `[status:done]` | Notes where `status` equals `done` |
| `[priority:>3]` | Notes where `priority` is greater than 3 |
| `date>=2024-01-01` | Notes modified after 2024-01-01 |

#### Content / Task Filters

| Syntax | Description |
|--------|-------------|
| `content:keyword` | Chunks containing a specific word in the body |
| `task:keyword` | Task items (`- [ ]`, `- [x]`) |
| `task-todo:keyword` | Incomplete tasks only |
| `task-done:keyword` | Completed tasks only |

#### Other Advanced Filters

| Syntax | Description |
|--------|-------------|
| `-keyword` | Exclude a specific word from results |
| `/regex/` | Regex pattern search |
| `match-case:keyword` | Case-sensitive match |

## 🚀 Usage

### Step 1: Installation

#### Manual Install
1. Download the latest `main.js`, `manifest.json`, `styles.css` from [Releases](https://github.com/highlow12/Obsidian-vault-llm/releases).
2. Copy them to your vault's plugin folder:
   ```
   <vault>/.obsidian/plugins/obsidian-vault-llm/
   ```
3. Obsidian → Settings → Community Plugins → Enable "Obsidian Vault LLM"

#### Build from Source
```bash
npm install
npm run build
```

Copy the generated `main.js`, `manifest.json`, `styles.css` to the plugin folder.

> **Requirements**: Obsidian 1.5.0+, Desktop only

---

### Step 2: Configuration

Obsidian → Settings → **OVL Settings** tab.

#### LLM API Settings

| Setting | Description |
|---------|-------------|
| API Provider | Gemini / Claude / OpenAI Compatible / Ollama / Custom |
| API URL | Chat endpoint URL (auto-filled when provider is selected) |
| API Key | Provider API key — **masked in UI** |
| Model | LLM model name |
| Title Model | Model for session title generation (falls back to main model) |
| System Prompt | System message included in all requests |
| Default Output Folder | Folder path for saving conversations |

**Google Gemini Example**
1. Select `Google Gemini` as provider
2. Enter your [Google AI Studio](https://aistudio.google.com/) API key
3. Click **Load Gemini Model List** → select model from dropdown

**Anthropic Claude Example**
1. Select `Claude (Anthropic)` as provider
2. Enter your [Anthropic Console](https://console.anthropic.com/) API key
3. Enter model name (e.g., `claude-sonnet-4-5`)

**Ollama (Local) Example**
1. Install [Ollama](https://ollama.com) and pull a model: `ollama pull llama3.1`
2. Select `Ollama (Local)` as provider (URL auto-filled)
3. Enter `llama3.1` in the model field

#### Vector Indexing Settings (for RAG)

| Setting | Default | Description |
|---------|---------|-------------|
| Enable Indexing | OFF | Enable vault indexing and vector search |
| Chunk Size | 400 | Max tokens per chunk |
| Chunk Overlap | 50 | Overlapping tokens between adjacent chunks |
| Top-K | 8 | Max results returned per search |
| Search Similarity Threshold | 0.03 | Minimum similarity for RAG results (0–1) |
| Save Similarity Threshold | 0.75 | Topic separation similarity threshold (0–1) |

---

### Step 3: Start Chatting

1. Click the **message icon** in the left ribbon or run **"Open OVL Chat"** from the command palette
2. Type a question and press **Send** or **Ctrl+Enter**

### Vault Indexing & RAG Search

1. Enable **Indexing** in settings
2. Click **Start Full Embedding** (first time, or after changing embedding model)
3. Use the **Vault Search Answer** button in the chat panel for RAG queries

### Saving & Topic Separation

Click the **Save** button in the chat panel header:
- The current conversation is saved as a markdown note
- Topic Separation AI automatically splits semantically different topics into separate notes (recommended: 4+ turns)

## 🙋 FAQ

**Q: Do I need a paid API?**
A: An LLM API key is required. Gemini has a free quota. Ollama is fully local and free.

**Q: Can I use it offline?**
A: Yes — with local embedding (HuggingFace) + Ollama, fully offline use is possible.

**Q: Is my data safe?**
A: Vault data is stored locally. Context is sent to the LLM API you configure. See your provider's privacy policy. The author makes no security guarantees.

**Q: How long does initial indexing take?**
A: ~1–2 minutes for 100 files, ~10–20 minutes for 1000 files (varies by embedding API speed).

**Q: Topic separation isn't working well.**
A: It requires 4+ turns and meaningful semantic differences between topics. Short or single-topic conversations will not be split.
